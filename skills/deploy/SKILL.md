# Deploy — 배포 워크플로우

## 개요

배포는 되돌리기 어려운 작업이므로 체계적 검증이 필수.
사전 검증 → 배포 실행 → 사후 확인의 3단계를 따른다.

## Phase 1: 사전 검증 (Pre-deploy)

배포 전 반드시 확인:

```
- [ ] 모든 테스트 통과 (전체 스위트)
- [ ] 빌드 성공 (프로덕션 빌드)
- [ ] 린트/타입체크 클린
- [ ] 환경변수 확인 (프로덕션 값 세팅)
- [ ] 마이그레이션 필요 여부 확인
- [ ] 변경사항 요약 준비
```

### 검증 명령 실행

```bash
# 프로젝트에 맞게 교체 <!-- TODO -->
npm run build          # 빌드
npm test               # 테스트
npm run lint           # 린트
npm run typecheck      # 타입체크
```

하나라도 실패하면 배포 진행 금지.

## Phase 2: 배포 실행

### 배포 전 확인

사용자에게 최종 확인:
```
배포 준비 완료:
- 환경: <production/staging>
- 브랜치: <branch-name>
- 변경사항: <요약>
- 마이그레이션: <있음/없음>

진행하시겠습니까?
```

### 배포 명령

```bash
# 프로젝트에 맞게 교체 <!-- TODO -->
# 예시:
git push origin main
# 또는
npm run deploy
# 또는
gh workflow run deploy.yml
```

## Phase 3: 사후 확인 (Post-deploy)

```
- [ ] 배포 성공 확인 (CI/CD 상태)
- [ ] 헬스체크 통과
- [ ] 핵심 기능 스모크 테스트
- [ ] 에러 모니터링 확인 (로그, 알림)
- [ ] 배포 결과 팀에 공유
```

## 롤백 계획

배포 전에 롤백 방법을 항상 준비:
```bash
# 프로젝트에 맞게 교체 <!-- TODO -->
# 예시:
git revert HEAD
# 또는
# 이전 배포 버전 재배포
```

## 금지 사항

- 테스트/빌드 실패 상태에서 배포
- 사용자 확인 없이 프로덕션 배포
- 롤백 계획 없이 배포
- 금요일 오후 배포 (팀 합의 없이)

## 규칙

- 배포는 항상 사용자의 명시적 승인 후 실행
- 모든 단계에서 증거 기반 확인
- 문제 발생 시 즉시 롤백, 원인 조사는 이후
